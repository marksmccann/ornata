The `lifecycle` option lets you run code at key points in a component's lifecycle.

## `mount`

`lifecycle.mount` runs once when the component is mounted.

```js
lifecycle: {
    mount() {
        console.log("Mounted");
    },
}
```

This is a good place for:

- starting timers
- registering observers
- connecting third-party integrations
- performing one-time setup logic
- injecting generated DOM elements
- deriving state defaults at startup

## `unmount`

`lifecycle.unmount` runs once when the component is disposed or unmounted.

```js
lifecycle: {
    unmount() {
        console.log("Cleaned up");
    },
}
```

This is a good place for:

- stopping timers
- disconnecting observers
- general teardown logic
- disconnecting from third-party integrations
- removing generated DOM elements

## A practical example

In this example, a value declared in data is stored for the life of the component instance so it can be created in `mount()` and cleaned up in `unmount()`.

<Aside>
This example also happens to demonstrate why `data` exists: some values need to persist across lifecycle hooks without becoming reactive state. See [Data](/ornata/guides/data/) for more information about that concept.
</Aside>

<Tabs syncKey="code-language">
  <TabItem label="JavaScript">

```js
const Clock = defineComponent({
    name: "Clock",
    data: {
        intervalId: null,
    },
    lifecycle: {
        mount() {
            this.data.intervalId = window.setInterval(() => {
                console.log("tick");
            }, 1000);
        },
        unmount() {
            if (this.data.intervalId !== null) {
                window.clearInterval(this.data.intervalId);
            }
        },
    },
});
```

## A good mental model

Think of `lifecycle` as a series of checkpoints in the life of a component. Each lifecycle method gives you a predictable place to respond to those moments.
