`render` is how Ornata turns component state back into DOM output inside the component's HTML contract. Each render callback returns a `RenderOptions` object for one resolved element or one item in a resolved element list.

## Basic shape

<Tabs syncKey="code-language">
  <TabItem label="JavaScript">

```js
render: {
    value() {
        return {
            text: String(this.state.count),
        };
    },
}
```

  </TabItem>
  <TabItem label="TypeScript">

```ts
interface CounterElements {
    value: HTMLElement
};

render: {
    value() {
        return {
            text: String(this.state.count),
        };
    },
}
```

  </TabItem>
</Tabs>

## Available render outputs

Render callbacks can return: `text`, `html`, `attributes`, `classes`, `style`, `dataset`, and/or `events`. These options can be combined in a single render result.

### `text`

Set `text` when you want to update `textContent`.

<Tabs syncKey="code-language">
  <TabItem label="JavaScript">

```js
render: {
    label() {
        return {
            text: this.state.label,
        };
    },
}
```

  </TabItem>
  <TabItem label="TypeScript">

```ts
interface ExampleElements {
    label: HTMLElement;
}

render: {
    label() {
        return {
            text: this.state.label,
        };
    },
}
```

  </TabItem>
</Tabs>

### `html`

Set `html` when you want to update `innerHTML`.

<Aside type="caution" title="Use trusted HTML only">
    `html` writes directly to `innerHTML`. Only use it with trusted content or
    content you have sanitized first. If you only need to show plain text,
    prefer `text` instead.
</Aside>

<Tabs syncKey="code-language">
  <TabItem label="JavaScript">

```js
render: {
    output() {
        return {
            html: `<strong>${this.state.message}</strong>`,
        };
    },
}
```

  </TabItem>
  <TabItem label="TypeScript">

```ts
interface ExampleElements {
    output: HTMLElement;
}

render: {
    output() {
        return {
            html: `<strong>${this.state.message}</strong>`,
        };
    },
}
```

  </TabItem>
</Tabs>

### `attributes`

Use `attributes` for standard and boolean attributes.

<Tabs syncKey="code-language">
  <TabItem label="JavaScript">

```js
render: {
    panel() {
        return {
            attributes: {
                hidden: !this.state.open,
                "aria-expanded": String(this.state.open),
            },
        };
    },
}
```

  </TabItem>
  <TabItem label="TypeScript">

```ts
interface ExampleElements {
    panel: HTMLElement;
}

render: {
    panel() {
        return {
            attributes: {
                hidden: !this.state.open,
                "aria-expanded": String(this.state.open),
            },
        };
    },
}
```

  </TabItem>
</Tabs>

Behavior summary:

- string values set the attribute
- `true` adds a boolean attribute
- `false` removes a boolean attribute
- `null` removes the attribute
- `undefined` leaves it unchanged

## Rendering lists with `index`

When the rendered element is an array, Ornata passes `index` in the render context.

<Tabs syncKey="code-language">
  <TabItem label="JavaScript">

```js
render: {
    items({ index }) {
        const currentIndex = index ?? 0;

        return {
            classes: {
                "is-active": currentIndex === this.state.activeIndex,
            },
        };
    },
}
```

  </TabItem>
  <TabItem label="TypeScript">

```ts
interface ExampleElements {
    items: HTMLElement[];
}

render: {
    items({ index }) {
        const currentIndex = index ?? 0;

        return {
            classes: {
                "is-active": currentIndex === this.state.activeIndex,
            },
        };
    },
}
```

  </TabItem>
</Tabs>

This is especially useful for:

- tabs
- menus
- repeated buttons
- repeated panels
- filtered or selected lists

## A good mental model

Render callbacks should describe DOM output, not general side effects or reusable business logic.

That usually means:

- `render` for visual and attribute updates
- `watch` for internal side effects and more complex DOM updates
- `methods` for named reusable actions and event handlers
- `computed` for derived state values
